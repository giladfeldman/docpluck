## ⚠️ Environment gotcha (cost us hours — don't repeat)

Long-running background commands die ~16–34 min in, when the user is away. **Two independent killers**, both diagnosed from event logs:

1. **Modern Standby (system S0 sleep).** Fixed this session via `powercfg /change standby-timeout-ac 0` (verified: no `Kernel-Power` 506 event through a 75-min idle afterward). Permanent fix (admin + reboot): `reg add HKLM\SYSTEM\CurrentControlSet\Control\Power /v PlatformAoAcOverride /t REG_DWORD /d 0 /f`.
2. **Claude Desktop suspends in-session `run_in_background` tasks** ~28 min after the user's last message. Host-app behavior — can't patch. Even a Windows Task Scheduler task died once.

**The fix that works:** `pytest-xdist` (added to `pyproject.toml [dev]`). Run the full suite parallel:
```
python -m pytest -q -n 10 --tb=short
```
~19 min on this 14-core box. One DOCX perf test (`test_docx_extraction_under_1s`) flakes under `-n10` CPU contention — re-run it sequentially to confirm (passes at ~1.01s); not a regression. For the corpus: `python scripts/verify_corpus.py` (~15 min, fits one window).
